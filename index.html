Drupal8 Fields Default markup

a suggestion for a markup structure for Drupals fields Field default markup - removing the divitis

Drupals fields come out in 4 different variations:

  1. Single value
  2. Single value with Label
  3. Multiple values
  4. Multiple values with Label

Single vs multiple


  <div class="field-single field-content">

  <div class="field-label">label</div>

1. Single value

The single value field with no label will have wrapper have .field-single & .field-content 


  <div class="field-single field-content ... field-label-hidden">
  Data single value no label 
  </div>
Data single value no label 
  .field-content{ color: red;}
  .field-single.field-content{ ... }

2 single value with Label

wrapper have .field-single .field-content is added to its field, so its seperated from the field-label, providing a sane structure.

Markup: 

<div class="field-single ... field-label-above">
  <div class="field-label">one field with label</div>
  <div class="field-content">one field with label</div>
</div>

Pro

  1. Removes the 3 layers of divs around a single value field with no label
  2. Removes the 2 layers of divs around a multiple value field with no label
  3. keeps the structure for a multiple value field, but dont add cruft to fields that dont need it

cons

  1. .field-single .field-content & .field-single.field-content
  2. .field-multiple .field-content & .field-multiple.field-content

a field will be outputtet 4 different ways, in the css that would mean that you would write
